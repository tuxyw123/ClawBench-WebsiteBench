# Repository agent instructions

For offline website-clone, source-evidence, Harbor benchmark, verifier,
calibration, corpus-expansion, or related project-management work, read and
follow `skills/build-offline-site-clone/SKILL.md` before acting.

Treat `project/plan.json` as the project-status source of truth. Validate it with
`clawbench-project validate`, preserve unrelated dirty work, and never mark a
benchmark release-ready without the declared Harbor and human-review evidence.
